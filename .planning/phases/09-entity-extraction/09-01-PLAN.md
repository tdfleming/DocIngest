```python
_model: TextEmbedding | None = None
_model_lock = threading.Lock()

def _get_model() -> TextEmbedding:
    global _model
    if _model is None:
        with _model_lock:
            if _model is None:
                _model = TextEmbedding(model_name=settings.fastembed_model)
    return _model
```

**Step 1: pyproject.toml** -- Add `"spacy>=3.7,<4"` to the dependencies list, after the `fastembed` entry (in the "Local embeddings" section or add a "# NLP" comment section).

**Step 2: src/docingest/services/entity_extraction.py** -- Create the service module with these exact functions, following the embedding.py pattern precisely:

1. **Module-level singleton** -- `_nlp: spacy.language.Language | None = None` and `_nlp_lock = threading.Lock()`. Use `structlog.get_logger()` for logging.

2. **`_get_nlp() -> spacy.language.Language`** -- Double-checked locking pattern identical to embedding.py's `_get_model()`. Log "loading spacy model" with model name on load, "spacy model loaded" on completion.

3. **`_SPACY_LABEL_MAP: dict[str, EntityType]`** -- Module-level dict mapping all 18 spaCy NER labels to EntityType values per the research:
   - PERSON->PERSON, ORG->ORGANIZATION, GPE/LOC/FAC->LOCATION, DATE/TIME->DATE
   - EVENT->EVENT, PRODUCT->PRODUCT, NORP/LAW/LANGUAGE/WORK_OF_ART->CONCEPT
   - CARDINAL/ORDINAL/MONEY/PERCENT/QUANTITY->OTHER

4. **`_map_spacy_label(label: str) -> EntityType`** -- Dict lookup with `EntityType.OTHER` as default fallback.

5. **`extract_entities(text: str) -> list[dict]`** -- Sync function. Call `_get_nlp()`, then lock around `nlp(text)` call (same as embedding.py locks around inference). Iterate `doc.ents`, build dicts with keys: `name` (stripped text), `entity_type` (via `_map_spacy_label`, use the EntityType value directly), `start_char`, `end_char`. Filter OUT entities where `entity_type == EntityType.OTHER` (per research recommendation to reduce noise). Cap at `settings.max_entities_per_chunk`. Return list.

6. **`extract_relationships(text: str, entities: list[dict]) -> list[dict]`** -- Sync function. Lock around `nlp(text)`. Build `entity_names` set (lowercased names from entities param). Traverse dependency tree: for each VERB token, collect subjects (children with dep `nsubj` or `nsubjpass`, expanded via `_get_span_text`) and objects (children with dep `dobj` or `attr`, expanded; also check `prep` children for `pobj` grandchildren for passive voice). For each subject-object pair, include only if BOTH `subj_text.lower()` and `obj_text.lower()` are in `entity_names` (per spec: "only include relationships where both source and target match an extracted entity"). Return list of dicts with keys: `source` (subject text), `target` (object text), `relation_type` (verb lemma), `description` (f"{subject} {verb lemma} {object}").

7. **`_get_span_text(token) -> str`** -- Helper to expand a token to include left compound/amod modifiers.

8. **`resolve_entity(name: str, entity_type: str, existing: list[dict], threshold: float | None = None) -> str | None`** -- Fuzzy match using `difflib.SequenceMatcher`. Default threshold from `settings.entity_confidence_threshold`. Require same `entity_type` match. Return best matching entity's `name` if ratio >= threshold, else None.

9. **Async wrappers** -- `extract_entities_async(text: str)` and `extract_relationships_async(text: str, entities: list[dict])` that use `asyncio.get_event_loop().run_in_executor(None, sync_fn, args)`. Note: `resolve_entity` does NOT need an async wrapper (it's pure Python string comparison, not CPU-bound spaCy work).

IMPORTANT: Do NOT create a class. Use module-level functions matching the embedding.py pattern exactly. All spaCy `nlp()` calls must be inside `with _nlp_lock:` blocks.
