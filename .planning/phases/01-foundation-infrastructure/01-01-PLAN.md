<task type="auto">
  <name>Task 2: Rewrite Azure-dependent modules to local alternatives</name>
  <files>src/docingest/config.py, src/docingest/db/blob.py, src/docingest/services/embedding.py</files>
  <action>
    **config.py:**
    - Remove all Azure settings (azure_blob_connection_string, azure_openai_endpoint, azure_openai_api_key, azure_openai_embedding_deployment, azure_openai_api_version)
    - Add MinIO settings: minio_endpoint (str, default "minio:9000"), minio_access_key (str, default "minioadmin"), minio_secret_key (str, default "minioadmin"), minio_secure (bool, default False), minio_bucket (str, default "docingest")
    - Add FastEmbed settings: fastembed_model (str, default "BAAI/bge-small-en-v1.5")
    - Change embedding_dimensions from 1536 to 384
    - Remove semantic_split_threshold (unused)
    - Remove folder watcher settings (watch_folder, watch_poll_interval)
    - Keep: mongodb, qdrant, redis, processing, rate limiting, API settings

    **blob.py:**
    Rewrite completely to use `minio` Python client (S3-compatible):
    - Import `from minio import Minio` and `from io import BytesIO`
    - Module-level `_client: Minio | None = None`
    - `get_blob_client() -> Minio`: Create Minio client using settings (minio_endpoint, minio_access_key, minio_secret_key, minio_secure). This is a sync client (Minio SDK is sync).
    - `close_blob() -> None`: Set _client to None (Minio client doesn't need async close)
    - `ensure_bucket(client: Minio) -> None`: Check `client.bucket_exists(settings.minio_bucket)`, create with `client.make_bucket()` if not
    - `upload_blob(client: Minio, tenant_id: str, blob_path: str, data: bytes, content_type: str) -> str`: Use `client.put_object(settings.minio_bucket, f"{tenant_id}/{blob_path}", BytesIO(data), len(data), content_type)`. Return the full path.
    - `download_blob(client: Minio, tenant_id: str, blob_path: str) -> bytes`: Use `client.get_object(settings.minio_bucket, f"{tenant_id}/{blob_path}")` and read response.
    - `delete_blob(client: Minio, tenant_id: str, blob_path: str) -> None`: Use `client.remove_object(settings.minio_bucket, f"{tenant_id}/{blob_path}")`
    - Key difference from Azure version: single bucket with tenant_id prefix paths (not per-tenant containers). Simpler and aligns with MinIO patterns.
    - Note: Minio Python client is synchronous. The functions can remain sync since they're called from async workers that can handle sync I/O. Remove `async` from function signatures.

    **embedding.py:**
    Rewrite completely to use `fastembed`:
    - Import `from fastembed import TextEmbedding`
    - Module-level `_model: TextEmbedding | None = None`
    - `_get_model() -> TextEmbedding`: Lazy-init TextEmbedding(model_name=settings.fastembed_model). This downloads the model on first use (~30MB for bge-small-en-v1.5).
    - `count_tokens(text: str) -> int`: Keep the ~4 chars/token approximation (good enough for chunking).
    - `embed_texts(texts: list[str]) -> list[list[float]]`: Use `list(model.embed(texts))` — FastEmbed returns a generator of numpy arrays. Convert each to list of floats. Handle batching internally (FastEmbed handles batching, but respect settings.embedding_batch_size for very large inputs by chunking the input list).
    - `embed_query(query: str) -> tuple[list[float], int]`: Use `list(model.query_embed(query))` — FastEmbed has a dedicated query_embed that prepends "query: " prefix for asymmetric models. Return (vector[0].tolist(), count_tokens(query)).
    - The fastembed functions are synchronous. Wrap in structlog for consistent logging.
    - Important: FastEmbed's embed() and query_embed() return generators of numpy arrays. Must convert to list[list[float]] via `.tolist()` on each array.
  </action>
