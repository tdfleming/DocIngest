  <name>Task 1: Fix search_chunks async/await and filter logic in qdrant.py</name>
  <files>src/docingest/db/qdrant.py</files>
  <action>
    Two bugs to fix in `search_chunks()`:

    **Bug 1 — Async/await precedence:**
    `return await client.query_points(...).points` accesses `.points` on the coroutine object (Python evaluates `.` before `await`). The coroutine doesn't have a `.points` attribute — this raises `AttributeError` at runtime.

    Fix: Separate the await and attribute access:
    ```python
    response = await client.query_points(
        collection_name=...,
        query=query_vector,
        limit=limit,
        query_filter=qdrant_filter,
        with_payload=True,
    )
    return response.points
    ```

    **Bug 2 — Filter AND/OR semantics:**
    Current code puts ALL conditions into `should` (OR) when >1 condition exists. This means `content_type=["pdf","docx"] AND source_ref="report"` becomes "match pdf OR docx OR report" — wrong.

    Correct semantics: OR within a single field's values, AND across different fields.

    Fix: Use `MatchAny` for list-valued filters (content_type), keep `MatchValue` for scalar filters (source_ref). Build one condition per field, combine with `must` (AND):
    ```python
    from qdrant_client.models import MatchAny
    # ...
    conditions = []
    for key, value in filters.items():
        if isinstance(value, list):
            conditions.append(FieldCondition(key=key, match=MatchAny(any=value)))
        else:
            conditions.append(FieldCondition(key=key, match=MatchValue(value=value)))
    qdrant_filter = Filter(must=conditions)
    ```

    This gives correct semantics: each field is one condition, all conditions must match (AND).
  </action>
