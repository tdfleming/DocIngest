       - At the TOP of the function (before temp file creation), check content_type:
         - If `content_type == "txt"`: decode raw_bytes to UTF-8 string, log, and return directly (no Docling). Plain text has no Markdown structure, so just return the raw text as-is.
         - If `content_type == "md"`: decode raw_bytes to UTF-8 string, log, and return directly (already Markdown).
       - The existing Docling path for PDF/DOCX/HTML remains unchanged.

    4. **workers/converter.py** — No changes needed. The converter worker already calls `convert_to_markdown()` which will now handle TXT/MD via the pass-through path. The rest of the pipeline (metadata extraction, blob upload, chunking enqueue) works identically regardless of format.
