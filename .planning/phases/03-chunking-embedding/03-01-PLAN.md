    Implement CHUNK-03: user can configure chunk_size, chunk_overlap, and strategy per upload.

    **1. config.py** — Add chunking strategy default:
    Add `chunking_strategy: str = "fixed"` to Settings class (after chunk_overlap_percent). This is the only supported strategy for v1; it validates against a known list.

    **2. api/routes/documents.py** — Add optional chunking params to upload endpoints:

    Add to `upload_document()` signature:
    ```python
    chunk_size: int | None = Query(None, ge=100, le=2000, description="Max tokens per chunk"),
    chunk_overlap: int | None = Query(None, ge=0, le=50, description="Overlap percentage"),
    chunking_strategy: str | None = Query(None, description="Chunking strategy: 'fixed'"),
    ```

    If `chunking_strategy` is provided and not "fixed", return 400 (unsupported strategy).

    Build a `chunking_config` dict from non-None values:
    ```python
    chunking_config = {}
    if chunk_size is not None:
        chunking_config["chunk_size"] = chunk_size
    if chunk_overlap is not None:
        chunking_config["chunk_overlap"] = chunk_overlap
    if chunking_strategy is not None:
        chunking_config["strategy"] = chunking_strategy
    ```

    Store `"chunking_config": chunking_config` in the document insert dict (alongside other fields). Only include the key if chunking_config is non-empty.

    Add the same params to `ingest_from_url()` — but as fields on `UrlIngestRequest`:
    ```python
    class UrlIngestRequest(BaseModel):
        url: HttpUrl
        force: bool = False
        chunk_size: int | None = Field(None, ge=100, le=2000)
        chunk_overlap: int | None = Field(None, ge=0, le=50)
        chunking_strategy: str | None = None
    ```

    Apply same config-building logic in `ingest_from_url()`.

    **3. services/chunking.py** — Accept optional per-doc overrides:

    Change `chunk_document()` signature to:
    ```python
    def chunk_document(
        markdown: str,
        *,
        max_tokens: int | None = None,
        overlap_percent: int | None = None,
    ) -> list[dict]:
    ```

    Inside, resolve values with fallback:
    ```python
    _max_tokens = max_tokens if max_tokens is not None else settings.chunk_max_tokens
    _overlap_pct = overlap_percent if overlap_percent is not None else settings.chunk_overlap_percent
    ```

    Use `_max_tokens` and `_overlap_pct` in the `_semantic_sub_split` call instead of reading settings directly.

    **4. workers/chunker.py** — Read per-doc config from MongoDB:

    After fetching the document (`doc = await get_document(...)`), extract chunking config:
    ```python
    chunk_cfg = doc.get("chunking_config", {})
    chunks = chunk_document(
        markdown,
        max_tokens=chunk_cfg.get("chunk_size"),
        overlap_percent=chunk_cfg.get("chunk_overlap"),
    )
    ```

    Import name note: `chunk_document` is imported from `docingest.services.chunking` — no conflict since the worker function is named `chunk_and_embed`.
