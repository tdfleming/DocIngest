<task type="auto">
  <name>Task 2: Add per-stage error handling to converter and chunker workers</name>
  <files>src/docingest/workers/converter.py, src/docingest/workers/chunker.py</files>
  <action>
    **Converter worker** (`workers/converter.py`):
    Replace the single broad `except Exception` with per-stage try/except blocks inside the main try. Each stage catches Exception, classifies the error, and re-raises a wrapper or calls update_document_status directly.

    Structure the `convert_document` function body as:
    ```
    try:
        # Stage: fetch document record
        doc = await db.documents.find_one(...)
        if not doc:
            log.error("document not found", doc_id=doc_id)
            await update_document_status(db, doc_id, DocumentStatus.FAILED,
                extra_fields={"error": "Document record not found in database",
                              "error_type": "not_found", "error_stage": "converting"})
            return

        # Stage: download raw file
        try:
            raw_bytes = download_blob(blob_client, tenant_id, doc["blob_path"])
        except Exception as e:
            ... update with error_type="download_error", error_stage="converting",
                error=f"Failed to download source file from storage: {e}"
            return

        # Stage: convert to markdown
        try:
            markdown = convert_to_markdown(raw_bytes, doc["content_type"], doc["source_ref"])
        except Exception as e:
            ... update with error_type="conversion_error", error_stage="converting",
                error=f"Failed to convert {doc['content_type']} to markdown: {e}"
            return

        # Stage: upload markdown
        try:
            upload_blob(blob_client, tenant_id, md_blob_path, markdown.encode("utf-8"), "text/markdown")
        except Exception as e:
            ... update with error_type="upload_error", error_stage="converting",
                error=f"Failed to upload converted markdown to storage: {e}"
            return

        # Stage: update status + enqueue (keep existing logic)
        ...
    except Exception as e:
        # Catch-all for unexpected errors
        log.error("conversion failed", doc_id=doc_id, error=str(e))
        await update_document_status(db, doc_id, DocumentStatus.FAILED,
            extra_fields={"error": str(e), "error_type": "unknown_error", "error_stage": "converting"})
    ```

    Log each stage-specific error with `log.error(...)` including doc_id, error_type, and error.

    **Chunker worker** (`workers/chunker.py`):
    Same pattern for `chunk_and_embed`:
    - `not_found` if doc missing (with status update, not silent return)
    - `download_error` for markdown download failure (error_stage="chunking")
    - `chunking_error` for chunk_document failure (error_stage="chunking")
    - `embedding_error` for embed_texts failure (error_stage="chunking")
    - `storage_error` for Qdrant upsert failure (error_stage="chunking")
    - `unknown_error` as catch-all (error_stage="chunking")

    Keep existing logic for empty chunks (status=COMPLETE, chunk_count=0).
    Keep existing ARQ WorkerSettings unchanged.
  </action>
  <verify>python -m py_compile src/docingest/workers/converter.py && python -m py_compile src/docingest/workers/chunker.py</verify>
