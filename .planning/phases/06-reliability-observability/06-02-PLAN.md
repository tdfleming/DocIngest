  <name>Task 1: Create logging config and add request logging middleware with trace IDs</name>
  <files>src/docingest/logging_config.py, src/docingest/api/middleware.py, src/docingest/api/app.py</files>
  <action>
    **Create `src/docingest/logging_config.py`:**
    - Define `configure_logging()` function that configures structlog for JSON output
    - Use `structlog.configure()` with:
      - `processors` chain: add_log_level, TimeStamper(fmt="iso"), StackInfoRenderer, format_exc_info, JSONRenderer
      - `wrapper_class`: structlog.make_filtering_bound_logger(logging.INFO)
      - `context_class`: dict
      - `logger_factory`: structlog.PrintLoggerFactory()
      - `cache_logger_on_first_use`: True
    - Import and configure stdlib logging to route through structlog as well (for third-party libs):
      ```python
      import logging
      logging.basicConfig(format="%(message)s", level=logging.INFO, handlers=[logging.StreamHandler()])
      ```
    - This ensures ALL log output (structlog + stdlib) is JSON-formatted

    **Update `src/docingest/api/middleware.py`:**
    - Keep existing `RateLimitHeaderMiddleware` unchanged
    - Add new `RequestLoggingMiddleware` (Starlette BaseHTTPMiddleware):
      - In dispatch: generate `trace_id = uuid.uuid4().hex[:16]`
      - Store `request.state.trace_id = trace_id`
      - Bind trace_id to structlog context: `structlog.contextvars.bind_contextvars(trace_id=trace_id)`
      - Record `start_time = time.monotonic()`
      - Call `response = await call_next(request)`
      - Calculate `duration_ms = round((time.monotonic() - start_time) * 1000, 1)`
      - Log: `log.info("request", method=request.method, path=request.url.path, status=response.status_code, duration_ms=duration_ms)`
      - Add `X-Trace-Id` header to response
      - Clear structlog contextvars: `structlog.contextvars.unbind_contextvars("trace_id")`
      - Return response
    - Import uuid, time, structlog

    **Update `src/docingest/api/app.py`:**
    - Import `configure_logging` from `docingest.logging_config`
    - Import `RequestLoggingMiddleware` from `docingest.api.middleware`
    - Call `configure_logging()` at module level (before app creation, so all loggers get JSON config)
    - Add `app.add_middleware(RequestLoggingMiddleware)` BEFORE the RateLimitHeaderMiddleware line (middleware executes in reverse order — RequestLogging wraps the outer layer)

    Do NOT use structlog.contextvars.merge_contextvars_context_processor — instead add `structlog.contextvars.merge_contextvars` to the processors chain in logging_config.py (this is the processor that reads from contextvars).
  </action>
