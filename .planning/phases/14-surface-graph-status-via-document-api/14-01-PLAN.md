<objective>
Surface the 4 graph-build progress fields (`graph_status`, `entity_count`, `relationship_count`, `graph_built_at`) through the document API. The graph-worker already writes these to MongoDB; they are stripped by `_doc_to_response` today. This plan adds them to `DocumentResponse`, updates `_doc_to_response`, declares `graph_built_at` on the `Document` model, and adds integration tests.

Purpose: Closes gap INT-02 from the v1.0 milestone audit. API consumers (including the future frontend) will be able to see per-document graph build status and progress.

Output:
- `src/docingest/models/document.py` with `graph_built_at` field declared
- `src/docingest/api/routes/documents.py` with extended `DocumentResponse` and `_doc_to_response`
- `tests/test_documents_graph_response.py` with 4+ integration tests
