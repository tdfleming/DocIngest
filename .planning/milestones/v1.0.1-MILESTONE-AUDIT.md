## Integration Verification (Phases 8-11)

Full cross-phase wiring report produced by `gsd-integration-checker`. Summary:

**Complete flows (4/6):**
- ✓ Ingest → Graph Build (chunker enqueue → graph worker → spaCy → graph_store). All imports and queue names align.
- ✓ Feature Gating (`GRAPH_RAG_ENABLED=false`): chunker skips enqueue, worker returns early, API returns 403, lifespan skips index creation.
- ✓ Community Rebuild: `POST /v1/graph/communities/rebuild` → `build_communities` → Leiden → TF-IDF → FastEmbed → `upsert_community` with hierarchy links.
- ✓ Multi-tenancy: no cross-tenant path; `tenant_id` flows correctly through every layer.
- ✓ Docker Wiring: graph-worker service, Dockerfile, spaCy download, deps all correct.

**Partial/broken flows (2/6):**
- **FLOW-04 (Reprocess cleanup):** `reprocess_document` in `documents.py:369` does NOT call `delete_doc_graph_data` — cleanup only happens inside the graph-worker job, creating a window where community rebuild can consume stale data.
- **FLOW-06 (Delete cleanup):** `delete_document_route` in `documents.py:343` does NOT call `delete_doc_graph_data` — orphaned entities/relationships accumulate permanently.

**Integration issues:**
- **INT-01:** `ensure_graph_indexes` called twice on startup (from `mongodb.ensure_indexes` AND from `app.py` lifespan). Idempotent but 10 redundant RPCs per boot.
- **INT-02:** `graph_status`/`entity_count`/`relationship_count` written to MongoDB but stripped by `_doc_to_response` → invisible to API consumers.

## Verdict

**v1.0 Milestone Audit (inclusive of Graph RAG extension): GAPS FOUND**

- v1.0 MVP (phases 1-7): passed in prior audit. Tech debt from that audit was closed in Phase 7.
- Graph RAG extension (phases 8-11): **cannot pass** in current state due to:
  1. **Functional gaps:** orphaned graph data on document delete, invisible graph status via API, stale-data window on reprocess.
  2. **Structural gaps:** no active `REQUIREMENTS.md`/`ROADMAP.md` defining the 25 extension REQ-IDs, missing `VERIFICATION.md` across all four extension phases, missing `requirements-completed` frontmatter on three SUMMARYs.
  3. **Policy conflict:** `PROJECT.md` Out-of-Scope still forbids what shipped.

Recommended path forward is NOT `/gsd:complete-milestone` — that would archive v1.0 in its current inconsistent state. Instead run `/gsd:plan-milestone-gaps` to close the functional and structural gaps, OR retroactively split the Graph RAG extension into a proper v1.1 milestone with formal scope.
